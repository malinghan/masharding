# v4.0 — SQL 解析与分片引擎

## 1. 版本目标

解决"如何自动从 SQL 中提取分片列的值"的问题。

实现 `ShardingEngine` 接口和 `StandardShardingEngine`，用 Druid SQL Parser 解析 SQL，自动提取逻辑表名和分片列值，结合 v3.0 的策略计算出目标库和目标表，返回包含物理 SQL 的 `ShardingResult`。从此不再需要手动构造 `params` Map。

---

## 2. 核心功能说明

### 类比

> 给中间件装上"眼睛"——能读懂 SQL，从中提取出"按哪个字段分片"以及"字段值是多少"，然后自动算出去哪个库哪张表。

### SQL 解析流程

```
engine.sharding("insert into user(id,name,age) values(?,?,?)", [3,"ma",20])
        │
        ▼
Druid SQLUtils.parseSingleMysqlStatement(sql)
        │
        ├─ INSERT → SQLInsertStatement
        │     提取列名列表: [id, name, age]
        │     与 args 对齐: {id:3, name:"ma", age:20}
        │
        └─ SELECT/UPDATE/DELETE → MySqlSchemaStatVisitor
              提取表名: user
              提取条件: WHERE id=3 → {id:3}
        │
        ▼
databaseStrategy.doSharding(["ds0","ds1"], "user", {id:3}) → "ds1"
tableStrategy.doSharding(["user0","user1","user2"], "user", {id:3}) → "user0"
        │
        ▼
将 SQL 中的逻辑表名 "user" 替换为物理表名 "user0"
        │
        ▼
ShardingResult(
  targetDataSourceName = "ds1",
  targetSqlStatement   = "insert into user0(id,name,age) values(?,?,?)"
)
```

### INSERT vs SELECT 解析差异

```
INSERT 解析：
  SQL: insert into user(id, name, age) values(?, ?, ?)
  args: [3, "ma", 20]
  ─────────────────────────────────────
  列名列表: [id, name, age]   ← 从 SQLInsertStatement.getColumns() 获取
  参数对齐: zip(列名, args)   ← {id:3, name:"ma", age:20}

SELECT/UPDATE/DELETE 解析：
  SQL: select * from user where id = ?
  args: [4]
  ─────────────────────────────────────
  表名: user                  ← MySqlSchemaStatVisitor.getTables()
  条件: id = ?                ← MySqlSchemaStatVisitor.getConditions()
  参数对齐: 按条件顺序与 args 对应 → {id:4}
```

## 4. 代码实现说明

### ShardingEngine.java

引擎接口，输入逻辑 SQL + 参数，输出分片结果。

```java
public interface ShardingEngine {

    /**
     * 执行分片计算
     *
     * @param sql  逻辑 SQL（含逻辑表名）
     * @param args MyBatis 传入的参数数组
     * @return 分片结果（目标数据源 + 物理 SQL）
     */
    ShardingResult sharding(String sql, Object[] args);
}
```

### StandardShardingEngine.java

核心实现，处理 INSERT 和 SELECT/UPDATE/DELETE 两类 SQL。

```java
public class StandardShardingEngine implements ShardingEngine {

    private final ShardingProperties properties;
    private final ShardingStrategy databaseStrategy;
    // key = 逻辑表名，value = 对应的表分片策略
    private final Map<String, ShardingStrategy> tableStrategies;

    public StandardShardingEngine(ShardingProperties properties) {
        this.properties = properties;

        // 初始化数据库分片策略
        ShardingProperties.StrategyProperties dbStrategyProps =
            properties.getDatabaseStrategy();
        this.databaseStrategy = new HashShardingStrategy(
            dbStrategyProps.getShardingColumn(),
            dbStrategyProps.getAlgorithmExpression()
        );

        // 初始化各逻辑表的表分片策略
        this.tableStrategies = new LinkedHashMap<>();
        for (Map.Entry<String, ShardingProperties.StrategyProperties> entry
                : properties.getTableStrategies().entrySet()) {
            tableStrategies.put(entry.getKey(), new HashShardingStrategy(
                entry.getValue().getShardingColumn(),
                entry.getValue().getAlgorithmExpression()
            ));
        }
    }

    @Override
    public ShardingResult sharding(String sql, Object[] args) {
        // 用 Druid 解析 SQL
        SQLStatement statement = SQLUtils.parseSingleMysqlStatement(sql);

        String logicTable;
        Map<String, Object> shardingParams;

        if (statement instanceof SQLInsertStatement) {
            // INSERT 语句：从列名列表和 args 对齐提取参数
            SQLInsertStatement insert = (SQLInsertStatement) statement;
            logicTable = insert.getTableName().getSimpleName().toLowerCase();
            shardingParams = extractInsertParams(insert, args);
        } else {
            // SELECT / UPDATE / DELETE：用 Visitor 提取表名和 WHERE 条件
            MySqlSchemaStatVisitor visitor = new MySqlSchemaStatVisitor();
            statement.accept(visitor);
            logicTable = visitor.getTables().keySet().iterator().next()
                .getName().toLowerCase();
            shardingParams = extractConditionParams(visitor, args);
        }

        // 获取可用数据源列表和物理表列表
        List<String> availableDataSources = new ArrayList<>(
            properties.getDatasources().keySet());
        List<String> availableTables = properties.getActualTables()
            .getOrDefault(logicTable, Collections.emptyList());

        // 计算目标数据源
        String targetDataSource = databaseStrategy.doSharding(
            availableDataSources, logicTable, shardingParams);

        // 计算目标物理表
        ShardingStrategy tableStrategy = tableStrategies.get(logicTable);
        String targetTable = tableStrategy.doSharding(
            availableTables, logicTable, shardingParams);

        // 将 SQL 中的逻辑表名替换为物理表名
        String targetSql = sql.replaceAll(
            "(?i)\\b" + logicTable + "\\b", targetTable);

        System.out.println("target db.table = " + targetDataSource + "." + targetTable);

        return new ShardingResult(targetDataSource, targetSql);
    }

    /**
     * 从 INSERT 语句提取分片参数
     * 将列名列表与 args 数组按位置对齐
     */
    private Map<String, Object> extractInsertParams(
            SQLInsertStatement insert, Object[] args) {
        Map<String, Object> params = new LinkedHashMap<>();
        List<SQLExpr> columns = insert.getColumns();
        for (int i = 0; i < columns.size() && i < args.length; i++) {
            String colName = ((SQLIdentifierExpr) columns.get(i))
                .getName().toLowerCase();
            params.put(colName, args[i]);
        }
        return params;
    }

    /**
     * 从 WHERE 条件提取分片参数
     * 按条件出现顺序与 args 数组对齐
     */
    private Map<String, Object> extractConditionParams(
            MySqlSchemaStatVisitor visitor, Object[] args) {
        Map<String, Object> params = new LinkedHashMap<>();
        int argIndex = 0;
        for (TableStat.Condition condition : visitor.getConditions()) {
            String colName = condition.getColumn().getName().toLowerCase();
            if (argIndex < args.length) {
                params.put(colName, args[argIndex++]);
            }
        }
        return params;
    }
}
```

```java

        // ... 同 v1.0，省略
    }

    public ShardingEngine shardingEngine(ShardingProperties properties) {
        return new StandardShardingEngine(properties);
    }
```

## 5. 测试流程

### 单元测试（无需启动 Spring）

```java
public class StandardShardingEngineTest {

    private StandardShardingEngine engine;

    @Before
    public void setUp() {
        ShardingProperties props = new ShardingProperties();
        // 配置数据源
        Map<String, ShardingProperties.DataSourceProperties> ds = new LinkedHashMap<>();
        ds.put("ds0", mockDsProps("db0")); ds.put("ds1", mockDsProps("db1"));
        props.setDatasources(ds);
        // 配置数据库策略
        props.setDatabaseStrategy(new ShardingProperties.StrategyProperties("id", "ds${id % 2}"));
        // 配置表策略
        Map<String, ShardingProperties.StrategyProperties> ts = new LinkedHashMap<>();
        ts.put("user", new ShardingProperties.StrategyProperties("id", "user${id % 3}"));
        props.setTableStrategies(ts);
        // 配置物理表列表
        props.setActualTables(Collections.singletonMap(
            "user", Arrays.asList("user0", "user1", "user2")));
        engine = new StandardShardingEngine(props);
    }

    @Test
    public void testInsert() {
        ShardingResult result = engine.sharding(
            "insert into user(id,name,age) values(?,?,?)",
            new Object[]{3, "ma", 20}
        );
        // id=3: ds${3%2}=ds1, user${3%3}=user0
        assertEquals("ds1", result.getTargetDataSourceName());
        assertTrue(result.getTargetSqlStatement().contains("user0"));
    }

    @Test
    public void testSelect() {
        ShardingResult result = engine.sharding(
            "select * from user where id=?",
            new Object[]{4}
        );
        // id=4: ds${4%2}=ds0, user${4%3}=user1
        assertEquals("ds0", result.getTargetDataSourceName());
        assertTrue(result.getTargetSqlStatement().contains("user1"));
    }
}
```

## 6. 版本总结

### 学到了什么

- Druid SQL Parser 提供了完整的 MySQL SQL 解析能力，`SQLUtils.parseSingleMysqlStatement()` 是入口。
- `MySqlSchemaStatVisitor` 是访问者模式的实现，遍历 AST 节点收集表名和 WHERE 条件，无需手动遍历语法树。
- INSERT 和 SELECT 的参数提取逻辑不同：INSERT 靠列名列表与 args 位置对齐；SELECT 靠 WHERE 条件顺序与 args 对齐。
- SQL 中逻辑表名替换为物理表名用正则 `replaceAll("(?i)\\b" + logicTable + "\\b", targetTable)` 实现，`\\b` 保证只替换完整单词。

### 与前版本对比

| 项目 | v3.0 | v4.0 |
|------|------|------|
| 分片参数来源 | 手动构造 Map | 自动从 SQL 解析 |
| SQL 改写 | 无 | 逻辑表名→物理表名 |
| 支持 SQL 类型 | 无 | INSERT/SELECT/UPDATE/DELETE |
| 依赖数据库 | 否 | 否（纯计算） |

### 局限性

- SQL 改写用正则替换，对复杂 SQL（子查询、别名）可能误替换。
- `MySqlSchemaStatVisitor` 提取条件时，复杂 WHERE（OR、IN、BETWEEN）的参数对齐可能不准确。
- 还没有与 MyBatis 集成，需要手动调用 `engine.sharding()`。

### 设计要点

`StandardShardingEngine` 是整个框架的"大脑"，它将 SQL 解析、策略计算、SQL 改写三个步骤串联成一个原子操作，输出的 `ShardingResult` 同时包含路由目标和改写后的 SQL，为后续 MyBatis 集成提供了完整的信息。
